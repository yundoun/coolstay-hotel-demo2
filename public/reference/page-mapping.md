# 페이지 매핑: 소노 → 꿀스테이

> 소노의 어떤 페이지를 참고하여 꿀스테이의 어떤 페이지를 만드는지 매핑.
> Planner는 이 매핑을 기반으로 계약서를 작성한다.

---

## 매핑 테이블

| # | 소노 레퍼런스 | 소노 URL | 꿀스테이 페이지 | 꿀스테이 URL | 변환 포인트 | 스크린샷 |
|---|-------------|----------|---------------|-------------|------------|---------|
| 1 | GNB + 푸터 | 전체 | **공통 레이아웃** | 전체 | 메뉴 2개로 축소 (호텔소개, 예약). 예약 위젯 하단 고정 | `home/home-1`, `home-5` |
| 2 | 메인 히어로 + 섹션 | `/` | **메인 랜딩** | `/` | 히어로+예약위젯+호텔 하이라이트. 소노의 New&Hot/Event 구조 차용 | `home/*` |
| 3 | Destinations | `/destinations` | **호텔 목록** | `/hotels` | 소노의 3열 카드 그리드 + 필터 구조 차용. 꿀스테이는 가격/등급 비교 추가 | `etc/destinations-*` |
| 4 | 개별 호텔 템플릿 | `/solbeach_sc` 등 | **호텔 상세** | `/hotels/[id]` | 히어로→소개→객실→부대시설→위치. 소노 로컬메뉴를 탭/섹션 네비로 변환 | `hotel/*` |
| 5 | 리조트 소개 | `/solbeach_sc/aboutus` | **호텔 소개 섹션** | `/hotels/[id]#about` | 호텔 상세 내 섹션으로 통합 | `etc/aboutus` |
| 6 | 객실 상세 | `/calm_gj/roomsviewall/detail/RM*` | **객실 상세 모달** | `/hotels/[id]?room=[roomId]` | 별도 페이지 대신 모달/시트. 스펙테이블+어메니티+예약CTA | `roomDetail/*` |
| 7 | 예약 진입 | `/reserve` | **예약 메인** | `/reservation` | 소노의 카테고리 카드 중 "객실"만. 꿀스테이는 호텔 선택부터 시작 | `reserve/reserve-main-*` |
| 8 | 객실 예약 캘린더 | `/reserve` (객실 클릭 후) | **예약 검색** | `/reservation` | 좌측 필터 + 캘린더 뷰 + 날짜별 가용현황 | `reserve/reserve-1` |
| 9 | Reservation 모달 | 날짜 클릭 시 | **객실 선택 모달** | `/reservation` (모달) | 필터 탭 + 객실 목록 + 예상금액 + 예약하기 CTA | `reserve/reserve-2-modal*` |
| 10 | 예약/결제하기 | `/reserve/room/payment` | **결제** | `/reservation/payment` | 2열: 이용자정보+결제방법 / 선택정보 사이드바 | `reserve/payment-*` |
| 11 | 결제 완료 | `/reserve/room/payment/ok` | **결제 완료** | `/reservation/complete` | 체크 아이콘 + 예약 요약 + CTA | `reserve/payment:ok-*` |
| 12 | 예약 조회 | `/mysono/pc/reserve/room` | **예약 조회** | `/my/reservations` | 검색+필터 + 2열(리스트/상세) | `reserve/:mysono:...-*` |
| 13 | 예약 취소 | `/mysono/pc/reserve/room/cancel` | **예약 취소** | `/my/reservations/[id]/cancel` | 예약 요약 + 취소금액/위약금/환불 + CTA | `reserve/cancel*` |

## 참고 vs 신규 분류

### 소노 참고 (구조·디자인 차용)
- **공통 레이아웃**: GNB 스타일, 예약 위젯 (하단 고정), Footer
- **메인 랜딩**: 히어로 슬라이더, 섹션 구성 (New&Hot, Event)
- **호텔 목록**: Destinations 페이지의 3열 카드 그리드 + 필터
- **호텔 상세**: 개별 호텔 템플릿 (히어로→Experience→New&Hot→소개)
- **객실 상세**: 스펙 테이블, Room Details 어메니티, 아코디언
- **예약 캘린더**: 좌측 필터 + 캘린더 뷰 + 예약원활/마감임박
- **Reservation 모달**: 필터 탭 + 객실 목록 + 예상금액
- **결제/완료/조회/취소**: 레이아웃 그대로 차용

### 꿀스테이 신규 (소노에 없는 기능)
- **호텔 목록 가격 비교**: 카드에 가격 표시 + 가격순 정렬 (중개 플랫폼 핵심)
- **호텔 등급 필터**: PREMIUM / CLASSIC / BOUTIQUE 등급별 필터

## 예약 위젯 동작 정리

> 📷 `roomFilter/roomFilter-1,2,3.png`

예약 위젯은 **하단 고정 (sticky bottom)** 바로, 여러 페이지에 공통 노출.

### 호텔/지역 선택 (roomFilter-1)
- 텍스트 입력 → **실시간 자동완성** 드롭다운
- 트리 구조: 상위 호텔 → 하위 지점
- 각 항목에 타입 태그 (`펫` / `리조트, 호텔`)
- 아이콘: 🏠 객실 안내 페이지 이동 / 📊 가이드맵 이동

### 날짜 선택 (roomFilter-2)
- **2개월 동시 표시** 캘린더 팝오버
- 체크인 클릭 → 체크아웃 클릭 (범위 선택)
- 과거 날짜 비활성

### 객실/인원 선택 (roomFilter-3)
- 객실 `[- N +]`, 대인 `[- N +]`, 소인 `[- N +]` 카운터
- 리조트/호텔 기준 인원 안내 텍스트
- `[적용]` 버튼으로 확정

## 사용자 플로우

```
[메인 랜딩 /]
    │
    ├── GNB "호텔 소개" 클릭
    │       │
    │       ▼
    │   [호텔 목록 /hotels]  ← Destinations 구조 차용
    │       │
    │       ├── 필터 (지역, 등급, 가격)
    │       │
    │       └── 호텔 카드 클릭
    │               │
    │               ▼
    │           [호텔 상세 /hotels/[id]]  ← 개별 호텔 템플릿 차용
    │               │
    │               ├── 소개 섹션 (aboutus 차용)
    │               ├── 객실 목록 → 객실 카드 클릭 → 객실 상세 모달
    │               ├── 부대시설 섹션
    │               ├── 위치 섹션
    │               │
    │               └── "예약하기" CTA ──┐
    │                                   │
    ├── GNB "예약" 클릭 ────────────────┘
    │       │
    │       ▼
    │   [예약 /reservation]
    │       │
    │       ├── 예약 위젯 (호텔 자동완성 + 캘린더 + 인원)
    │       ├── 캘린더 뷰 (날짜별 예약원활/마감임박)
    │       ├── 날짜 클릭 → Reservation 모달
    │       │     ├── 필터 탭 (숙소유형/객실종류/전망/객실내부)
    │       │     ├── 객실 목록 + 상태 배지
    │       │     └── 예상금액 + "예약하기"
    │       │
    │       └── "예약하기" 클릭
    │               │
    │               ▼
    │           [결제 /reservation/payment]
    │               │
    │               ├── 이용자 정보 + 할인 혜택 + 결제방법
    │               ├── 선택 정보 사이드바
    │               ├── 이용약관 동의
    │               └── "예약/결제하기" → 결제 완료
    │                                       │
    │                                       ▼
    │                               [결제 완료 /reservation/complete]
    │                                       │
    │                                       └── "예약내역 확인" 클릭
    │                                               │
    │                                               ▼
    └── ──────────────────────────── [예약 조회 /my/reservations]
                                            │
                                            ├── 예약 검색 + 필터
                                            ├── 예약 카드 → 상세 보기
                                            └── "예약취소" 클릭
                                                    │
                                                    ▼
                                            [예약 취소 /my/reservations/[id]/cancel]
                                                    │
                                                    ├── 취소금액 / 위약금 / 환불금액
                                                    └── "예약/결제 취소" CTA
```
